GENERAL MECHANICS
ROM-EXTRACTED DATA ONLY
Data source notice: This wiki is built from data tables extracted directly from the ROM binary (Final Fantasy Legend, The (USA).gb). Only information derived from the extracted data files is presented here. Mechanics that have not yet been extracted from the ROM — including damage formulas, turn order logic, status effect rules, flee calculations, equipment slot counts, and elemental interactions — are omitted rather than inferred.

CHARACTER CLASSES

The ROM contains three character class identifiers. Player-class entries appear at monster indices 173–188 in the creature table. The three classes are distinct enough to occupy separate index blocks:

HUMAN

Equipment-focused class. Stats and mechanics not fully extracted from ROM.

→ Humans page
MUTANT

Random-growth class. Stats and mechanics not fully extracted from ROM.

→ Mutants page
MONSTER

Transformation class. Meat transformation table extracted (25 classes × member weights). Trigger conditions and meat slot count not yet extracted from ROM.

→ Monsters page

BOSS ROSTER (ROM DATA)

Final bosses are the last entries in the 200-monster table. Stats extracted directly from 0x1AAE8 (STR/DEF/AGI/MANA) and 0x1B254 (HP). Values that exceed 99 are shown as-is — the display cap of 99 observed in-game is a rendering limit, not the stored value.

ROM IDNAMEHPSTRDEFAGIMANAGOLD

Indices 189–199. HP from 0x1B254 (2-byte LE, verified). Gold from BCD table at 0x1B2A4. Values shown are raw stored integers; in-game display caps stats at 99.

ABILITY TYPE CODES (ROM DATA)

Each entry in the ability table at 0x14640 (252 entries × 8 bytes) encodes a type byte at position 7 and a prefix byte at position 0. These are the observed values and their classifications:

BYTE VALUEPOSITIONCLASSIFICATION
0x01, 0x03Type (byte 7)Usable item
